## ✍️ 怎么改内容(核心)

### 1. 发一篇新博文 📝

去 `src/content/blog/` 文件夹,**新建一个文件**,名字随便起(用英文,比如 `my-trip.md`,结尾必须是 `.md`)。

把下面这段**原样复制**进去,然后改成你的内容:

```markdown
---
title: "我的文章标题"
date: 2026-06-13
tags: ["随笔", "摄影"]
excerpt: "这句话会显示在文章列表里,当作摘要。"
foot: "结尾的一句斜体小字(不想要就把这一行删掉)"
---

正文从这里开始写,想写什么写什么。

中间空一行,就是一个新段落。

想让某些字**加粗**,就用两个星号包起来。

想分点列表:
- 第一点
- 第二点
- 第三点

想插入一张图:把图放进 public/assets/ 文件夹,然后这样写:
![图片说明](/assets/你的图片名.jpg)
```

保存后,网站**自动多出这篇文章**,排版、展开动画全自动套好。文章会按日期自动排序,最新的在最上面。

> **格式说明(最上面 `---` 之间的部分):**
> - `title` —— 文章标题(必填)
> - `date` —— 日期,格式 `年-月-日`(必填,决定排序)
> - `tags` —— 标签,用方括号和引号,可以写多个(必填)
> - `excerpt` —— 列表里显示的摘要(必填)
> - `foot` —— 结尾斜体小字(可选,不要就删掉这行)

### 2. 改「关于」那段自我介绍

打开 `src/data/profile.ts`,找到 `about = [` 那部分,改中间引号里的文字。
每一段用引号包起来,段与段之间用逗号隔开。

### 3. 改名字 / 头衔 / 一句话简介 / 社交链接

打开 `src/data/profile.ts`,最上面:
- `name` —— 左上角的大名字
- `title` —— 名字下面的头衔(独立开发者 · 摄影师 · 学生)
- `tagline` —— 再下面那段一句话简介
- `socials` 里的 `github` / `instagram` / `email` —— 填上你的链接;**留空 `''` 那个图标就不显示**
- `defaultPalette` —— 网站默认配色(11 选 1,名字在注释里列着)
- `defaultTheme` —— 默认深色 `dark` 还是浅色 `light`

### 4. 加 / 改「项目」卡片

打开 `src/data/projects.ts`,里面是一个个 `{ ... }` 块,每块是一个项目。
**复制一整块**,改成新项目的信息:
- `name` —— 项目名
- `href` —— 点击跳转的链接
- `repo` —— 填 `"用户名/仓库名"`,网站会**自动显示实时 star 数**(你 GitHub 涨星,网站跟着变)
- `stars` —— 兜底星数(网络抓取失败时显示它)
- `logo` —— 缩略图,把图放进 `public/assets/`,这里填 `/assets/图片名`
- `tags` —— 技术标签

不要项目了?把整个 `{ ... }` 块(连同后面的逗号)删掉。

### 5. 加 / 改「站点」卡片

打开 `src/data/sites.ts`,用法和项目一样,复制 `{ ... }` 块改内容。

### 6. 加 / 改「历程」

打开 `src/data/experiences.ts`,复制 `{ ... }` 块改内容:
- `meta` —— 时间(如 `2024 — 至今`)
- `title` —— 标题
- `desc` —— 描述
- `tags` —— 标签

### 7. 换图片

把图片文件丢进 `public/assets/` 文件夹,然后在需要的地方引用 `/assets/图片名`。

## 🚀 发布到线上(让别人看到)

改完内容、本地预览满意之后,要"发布"才会更新到 https://blog.loven7.com。

**最简单的方式**——双击运行 `site` 文件夹里的:

```
发布.bat
```

(这个脚本我帮你做好了,双击它会自动构建 + 上传到 Cloudflare,等一两分钟就更新了。)

**或者手动两条命令**(在 `site` 目录的终端里):
```
npm run build
npx wrangler pages deploy dist --project-name=linminheng-blog --branch=main
```

发布成功后,等 1-2 分钟,刷新 https://blog.loven7.com 就能看到新内容。

## ⚠️ 几个注意点

1. **改 `.ts` 文件时**,注意别删掉引号 `'`、逗号 `,`、花括号 `{ }`——它们是格式的一部分,删错了网站会报错。照着已有的样子改最安全。
2. **博文的 `.md` 文件**,最上面 `---` 之间的格式区不能乱,正文区随便写。
3. **每次改完先本地预览**(`npm run dev`)确认没问题,再发布到线上。
4. **拿不准的时候**,先复制一份原文件备份,改坏了能还原。

---

## 🆘 出问题了怎么办

- **本地预览打不开 / 报错**:把终端里的红色报错信息截图,大概率是某个 `.ts` 文件的引号或逗号删错了。
- **发布后线上没更新**:等 2-3 分钟再刷新;还不行就清浏览器缓存(Ctrl+F5)。
- **彻底搞乱了**:GitHub 上有完整备份,所有历史版本都能找回来。

## 🗂 文件结构速查

```
site/
├─ src/
│  ├─ content/blog/      ← 博文,一篇一个 .md 文件【最常改】
│  ├─ data/
│  │   ├─ profile.ts     ← 名字/头衔/简介/社交/关于
│  │   ├─ projects.ts    ← 项目卡片
│  │   ├─ sites.ts       ← 站点卡片
│  │   └─ experiences.ts ← 历程
│  ├─ layouts/Base.astro ← 整体外壳(配色/动画/脚本,别动)
│  ├─ styles/global.css  ← 样式(别动)
│  └─ pages/index.astro  ← 首页组装(别动)
└─ public/assets/        ← 图片放这里
```

**带【最常改】的是你日常会碰的;标"别动"的是设计部分,改坏了会影响整站外观。**
