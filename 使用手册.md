# 一方天地 · 使用手册

这是一个静态个人博客。内容保存在仓库文件里,由 Astro 构建成静态页面,再部署到 Cloudflare Pages。

这套方式没有后台、没有数据库、没有线上写接口。文章和个人资料都能被 Git 记录、回滚和备份,长期维护更稳。

## 网站位置

- 本地源文件:`D:\files\Aitest4\Blog\site\`
- 线上地址:https://blog.loven7.com
- GitHub 仓库:https://github.com/Lur1N77777/linminheng-blog

## 本地预览

在 `site` 目录运行:

```bash
npm run dev
```

打开终端显示的地址,通常是:

```txt
http://localhost:4321
```

保存文件后浏览器会自动刷新。结束预览时在终端按 `Ctrl + C`。

也可以直接双击:

```txt
本地预览.bat
```

### 最省事的网页管理方式

项目里预留了一个 Git CMS:

```txt
/admin/
```

它不是数据库后台,而是一个网页表单编辑器。它编辑 Markdown/JSON 文件,保存时提交到 GitHub,再由自动部署更新网站。

后台入口有一层管理员密码,当前临时密码是:

```txt
loven7-admin
```

这层密码用于挡住普通访客。真正能不能保存内容,仍然取决于 GitHub 登录和仓库权限。

第一次使用需要在 GitHub 上确认登录和仓库授权。这个方式适合在外面用浏览器管理博文、项目、站点和个人资料。

### 写新文章

如果在自己的电脑上,推荐运行:

```bash
npm run new:post
```

按提示输入标题、标签、摘要,脚本会自动在 `src/content/blog/` 里生成 Markdown 文件。然后只需要写正文。

也可以直接双击:

```txt
新建博文.bat
```

### 在外面快速新建文章

如果人在外面,没有本地环境,不用复制模板。

打开 GitHub 仓库后:

1. 点 `Actions`
2. 选择 `New blog post`
3. 点 `Run workflow`
4. 填标题、标签、摘要
5. 提交后 GitHub 会自动创建一篇 Markdown 草稿

草稿默认 `draft: true`,不会显示在首页。写完正文后把它改成:

```txt
draft: false
```

再提交一次,网站就会自动发布。

文章放在:

```txt
src/content/blog/
```

一篇文章一个 `.md` 文件。建议用英文或日期命名,例如:

```txt
2026-06-19-blog-redesign.md
```

模板:

```markdown
---
title: "文章标题"
date: 2026-06-19
updated: 2026-06-19
tags: ["博客", "独立开发"]
excerpt: "文章列表里显示的摘要。"
draft: false
foot: "可选的结尾小字"
---

正文从这里开始。
```

常用字段:

- `title`:文章标题
- `date`:发布日期
- `updated`:最后更新日期,可选
- `tags`:标签
- `excerpt`:摘要
- `draft`:是否草稿。`true` 不会出现在首页
- `foot`:文章结尾的小字,可选
- `cover`:封面图路径,可选

### 在外面怎么改已有内容

不用构建,也不用上传 `dist`。

在外面的电脑、平板或手机上,直接打开 GitHub 仓库:

```txt
https://github.com/Lur1N77777/linminheng-blog
```

常见操作:

- 写文章:进入 `src/content/blog/`,新建或编辑 `.md` 文件
- 改个人信息:编辑 `src/data/profile.ts`
- 改项目:编辑 `src/data/projects.ts`
- 改站点:编辑 `src/data/sites.ts`
- 改历程:编辑 `src/data/experiences.ts`

改完点 GitHub 页面上的 `Commit changes`。提交到 `main` 分支后,GitHub Actions 会自动构建并部署到 Cloudflare Pages。

### 改个人信息

打开:

```txt
src/data/profile.ts
```

可以修改名字、头衔、简介、页脚、社交链接、默认配色和明暗主题。

### 改项目

本机推荐双击:

```txt
新建项目.bat
```

或者运行:

```bash
npm run new:project
```

也可以手动打开:

```txt
src/data/content/projects.json
```

`repo` 填 GitHub 仓库名,页面会在浏览器里同步 star 数。`stars` 是同步失败时显示的兜底数字。

### 改站点

本机推荐双击:

```txt
新建站点.bat
```

或者运行:

```bash
npm run new:site
```

也可以手动打开:

```txt
src/data/content/sites.json
```

这里维护你在网上的其他站点入口。

### 改历程

打开:

```txt
src/data/content/experiences.json
```

每一项包含时间、标题、描述、标签和可选链接。

### 换图片

图片放在:

```txt
public/assets/
```

引用时使用:

```txt
/assets/图片名.png
```

## 发布

日常推荐方式:

```bash
git add .
git commit -m "更新博客内容"
git push
```

推送到 `main` 后,GitHub Actions 会自动构建并发布。

本地手动检查:

```bash
npm run build
```

需要手动发布时,可以双击:

```txt
发布.bat
```

也可以手动运行:

```bash
npx wrangler pages deploy dist --project-name=linminheng-blog --branch=main --commit-dirty=true
```

自动部署需要 GitHub 仓库里配置 `CLOUDFLARE_API_TOKEN` 这个 secret。

## 文件结构

```txt
site/
├─ src/
│  ├─ content/blog/      # 博文 Markdown
│  ├─ data/              # 个人信息、项目、站点、历程
│  ├─ layouts/Base.astro # 页面外壳、主题切换和交互脚本
│  ├─ pages/index.astro  # 首页
│  └─ styles/global.css  # 全站样式
└─ public/assets/        # 图片和 logo
```

## 出问题时

- 构建失败:优先检查 Markdown 头部字段、日期格式、数组逗号
- 页面没更新:先确认本地 `npm run build` 成功,再重新发布
- 想回到旧版本:用 Git 历史回滚
